The workshop is about Relevance (Pseudo) Models vs IR models. You'll learn how to design and implement a pseudo-relevance model for a given initial query; select and implement baseline models (e.g., a BM25-based information retrieval model or the model you developed in Week 9); and evaluate the effectiveness of the proposed approach using appropriate metrics.

Solutions - Operations Manual

For Task1

The “PRM_bm25.py” file is designed for subtasks (1) to (3). It produces two files:

PRModel_R102.dat (the Training_set document ranking) and

PTraining_benchmark.txt (Generated relevance judgements by using query q="Convicts repeat offenders").

The “PRM_training_bm25_wk10.py” file extends Week 9 Task 1 solution. It uses Training_set and the generated relevance judgements “PTraining_benchmark.txt”. The output is “PModel_w5_R102.dat” (a set of features).

The “PRM_test_eval_bm25_wk10.py” file extends Week 9 Solutions for Task 2 and Task 3. It uses the features, “PModel_w5_R102.dat”, selected by “PRM_training_bm25_wk10.py” to rank documents in Test_set and saves in file “PR102_test_ranks.dat”. At last, it tests the ranking results as we did in Week 9.

For Task 2

The “IRM_bm25.py” file is a BM25 based IR model. It ranks documents in Test_set folder directly (Note that it does not require any training) and saves the ranking result in “IRModel_R102.dat”.

The “IRM_test_eval_bm25_wk11.py” file evaluates the ranking result “IRModel_R102.dat” against the true relevance judgements “Test_benchmark.txt”.

Other .py files:
